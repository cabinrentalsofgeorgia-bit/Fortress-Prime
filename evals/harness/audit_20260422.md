# Eval Harness Audit — 2026-04-22

## Current Metrics (run_eval.py)

### 1. `similarity_mean`
**What it measures:** Mean cosine similarity between the model's output text and the gold teacher response, computed with `all-MiniLM-L6-v2` embeddings. Reported overall and per domain.

**How it works:** Both strings are encoded as 384-dim sentence vectors; cosine similarity ranges 0–1. A score of 0.7+ roughly corresponds to semantically related responses; 0.5 is weak partial overlap.

**What it misses:**
- **Wrong citation, similar prose (legal/C):** If the model cites the wrong OCGA section but writes a plausible paraphrase, cosine similarity can be 0.6–0.8 even though the answer is factually wrong. The embedding model cannot distinguish "Under O.C.G.A. § 9-11-12(b)(6)" from "Under O.C.G.A. § 9-11-8(a)" if surrounding prose is similar.
- **Section header mismatch (legal/B):** Pattern B gold answers are the section headers of a specific court opinion (e.g., "Statutory Scheme", "Summary Judgment"). These are short, idiosyncratic strings. Any topic phrasing variation tanks similarity even when the concepts are correct. A model that correctly identifies the issues but uses different phrasing scores ~0.5.
- **Holding vs. dictum confusion (legal/A):** The model may reproduce the facts accurately but state the wrong legal outcome. Embedding similarity of the full text can still be high.
- **Rule number errors (legal/C, rules-based):** "Under GA Supreme Court Rule 4" vs "Rule 14" — similar embedding, wrong rule.
- **Hallucinated citations:** The model can invent a plausible-sounding OCGA citation or case name. Similarity with gold remains high if prose matches.

### 2. `validity_rate`
**What it measures:** Binary check per record — output is valid if: non-empty, length between 10% and 1000% of teacher length, special character ratio < 50%, no repeated n-gram pattern.

**What it misses:**
- A one-sentence response is "valid" even if it answers a different question.
- Structurally correct but factually empty output (e.g., "The applicable Georgia statute is not available") passes.
- Does not check format compliance (numbered list for B, "Under [citation]:" prefix for C).

### 3. `regression_count`
**What it measures:** Count of records where the student output is INVALID (fails validity check) but the teacher output IS valid. Per-domain.

**What it misses:**
- A response that is valid (not empty/gibberish) but factually wrong is not counted as a regression.
- Severity-blind: one completely wrong but well-formed answer counts the same as zero regressions.

## Correct Task Definitions Per Domain

| Domain | Pattern | Task Description | Gold Answer Format |
|--------|---------|------------------|--------------------|
| legal/A | A | Given insurance case facts, provide applicable legal analysis | Paragraph with holding + reasoning |
| legal/B | B | Identify key legal issues raised in insurance dispute (from case text) | Numbered list of section header topics |
| legal/C | C | Identify Georgia legal authority supporting a proposition | "Under [citation]: [proposition with citation filled]" |
| legal/D | D | Predict likely outcome given GA precedent | Paragraph citing matching precedent |
| legal/E | E | General GA legal reasoning | Variable |

**Critical note on legal/B:** This domain is NOT statute lookup. The task extracts section headers from a court opinion. The correct metric is topic recall against the gold section list, not statute faithfulness.

**Critical note on legal/C:** Two sub-types exist in the eval:
- Case citation C (from scripted.jsonl): cites GA case law ("Under Smith v. Jones, 123 Ga. 456:")
- Rules citation C (from court rules expansion): cites GA court procedural rules ("Under GA Supreme Court Rule N")

## Failure Modes Not Caught by Current Metrics

| Failure Mode | Domain | Undetected By |
|---|---|---|
| Wrong OCGA section number | C | similarity_mean |
| Wrong rule number | C | similarity_mean |
| Hallucinated case citation | A, C, D | all current metrics |
| Different section header phrasing (same concept) | B | similarity_mean (penalises correct answers) |
| Correct holding, wrong court level | A | similarity_mean |
| Missing required procedural element | C | similarity_mean |
| Paraphrase drift from cited section | B (if statute-based) | all current metrics |
| Citation in output not in GA legal corpus | A, C, D | all current metrics |

## New Metrics Implemented in run_eval_v2.py

### A. Citation Extraction Accuracy (`citation_f1`, legal/C primary)
Extracts all citations from output and gold; computes precision/recall/F1 on normalized citation tokens. Parser handles OCGA § variants, GA case citations, USCR rule references.

### B. Topic Recall/Precision/F1 (`topic_recall`, `topic_precision`, `topic_f1`, legal/B primary)
Parses numbered list items from output and gold; computes semantic match rate using MiniLM at threshold 0.7. Correctly scores outputs that use different phrasing for the same concept.

### C. Holding Term Overlap (`holding_overlap`, legal/A)
Extracts holding/affirmance/reversal statement; computes ROUGE-L and key legal term overlap between output and gold.

### D. Format Compliance (`format_ok`, per-domain)
- B: output is a numbered list
- C: output begins with "Under [something]:"
- A: output is prose (not a list)

### E. Hallucination Rate (`hallucination_rate`, all domains)
Fraction of citations in the output that do not match the expected citation format for GA law. Proxy for hallucinated authorities.

### F. Per-sample storage (`--save-outputs`)
When enabled, stores per-sample model output in `outputs.jsonl` alongside `metrics.json`. Required for Phase 2 failure mode analysis.
